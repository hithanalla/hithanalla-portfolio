Overview

When you care enough to audit a product you didn't build.

Reclips AI is a creative tool that lets users generate and edit AI-powered images and video. As a product consultant, I came in from the outside — no internal context, no prior relationship with the codebase — and mapped the full user experience to find where people were getting stuck.

This is the kind of work that sits at the intersection of product and marketing: understanding the user deeply enough to know why they leave before they convert, and translating that into concrete design recommendations a team can actually act on.

The brief: Identify friction points in the image prompt workflow, map the full user decision tree, and deliver actionable redesign recommendations — without touching a single line of code.

Process

How I approached the audit.

Product consulting from the outside means you only have one superpower: you see the product the way a new user does. I leaned into that.

01

Mapped every user decision point in Figma

Built a full UX audit map covering what users can click, what prompts help them choose, what content is available on the website, and where they hit dead ends. Colour-coded by node type: user actions, system responses, friction points, and outcomes.

02

Identified the core use case workflows

Mapped three primary workflows — creating a short AI film, making a shampoo ad for a new product, and creating a single AI image — to understand how different user goals led to completely different navigation paths through the same product.

03

Documented UI iterations with design rationale

Produced multiple desktop UI versions (2.1 through 2.4) showing how the image creation page should evolve — from the initial layout to a cleaner edit page — with annotated reasoning for each change.

04

Delivered clear, actionable recommendations

Every insight came with a specific recommendation the engineering or design team could act on immediately — not vague "improve the UX" suggestions, but "here's the exact navigation flow that's confusing users and here's how to fix it."

Key Findings

What I found — and what I recommended.

The image prompt interface had two layers of friction: users didn't know what they could make, and once they started, they didn't know how to get back or change direction. The navigation assumed familiarity the product hadn't earned yet.

Pain points found

Too many options shown at once — users paralysed before starting
Image prompt input unclear — users didn't know what level of detail to give
No clear path from image creation to video creation
Edit page felt disconnected from creation flow
Mobile navigation experience significantly worse than desktop

Recommendations made

Lead with use-case prompts, not feature menus — guide the first decision
Add inline prompt examples and guidance at the input stage
Build a clear creation → edit → export linear flow
Unify the edit page visual language with the creation interface
Simplify mobile to one primary action per screen

Reflection

What this taught me about product.

Coming in as an outsider is actually a structural advantage in UX consulting. The team had been staring at the product for months — they'd stopped seeing what new users see. My job was to be the first-time user, map that experience honestly, and translate it into something actionable.

The biggest insight wasn't a design fix — it was a positioning problem. The product was leading with its features when it should have been leading with user goals. That's a marketing problem as much as a design problem, and it reinforced how inseparable product and GTM really are.
